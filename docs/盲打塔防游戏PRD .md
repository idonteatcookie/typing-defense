## 1. 产品概述

### 1.1 产品简介

「盲打防线」是一款结合**打字练习**与**塔防策略**的 PC 网页 2D 游戏。玩家通过快速准确地输入字母来操控终点炮塔发射子弹消灭怪物，同时使用击杀怪物获得的金币购买并布置防御塔，在怪物抵达终点之前将其全部消灭。

游戏以"寓教于乐"为核心设计理念，让玩家在享受塔防游戏乐趣的同时，潜移默化地提升盲打速度和准确率。

### 1.2 产品定位

- **产品类型**：休闲益智 + 塔防策略
- **平台**：PC 网页浏览器
- **玩法模式**：单机 PVE 关卡制
- **目标时长**：单局 3-10 分钟

### 1.3 产品特色

| 特色 | 描述 |
|------|------|
| 打字+塔防双玩法 | 创新融合打字练习与塔防策略，两种玩法相互增益 |
| 多维度策略深度 | 塔的选择、放置位置、打字节奏共同影响胜负 |
| 渐进式难度曲线 | 从单字母到多字母组合，适配不同水平玩家 |
| 即时正反馈 | 每一次正确输入都有视觉+听觉反馈，成就感强 |
| 低门槛易上手 | 只需键盘输入，无需复杂操作教学 |

## 2. 目标用户与使用场景

### 2.1 目标用户画像

| 用户类型 | 特征 | 核心需求 |
|---------|------|----------|
| **学生群体** | 12-22岁，需要练习打字，喜欢游戏 | 有趣地提升打字速度 |
| **职场新人** | 22-30岁，想提高办公效率 | 碎片化时间练习盲打 |
| **休闲玩家** | 各年龄段，喜欢轻度策略游戏 | 轻松有趣的游戏体验 |
| **打字初学者** | 零基础或打字较慢 | 循序渐进的打字训练 |

### 2.2 使用场景

- **场景一**：午休/课间，打开网页玩 2-3 关，放松同时练打字
- **场景二**：每天花 10-15 分钟，系统性提升打字速度
- **场景三**：周末休闲，挑战更高难度关卡，追求通关成就

## 3. 产品目标与核心价值

### 3.1 产品目标

| 阶段 | 目标 | 关键指标 |
|------|------|----------|
| **MVP 阶段** | 验证核心玩法可行性 | 核心循环可玩性、留存率 |

3. **效率价值**：碎片化时间即可获得练习效果

## 5. 游戏核心玩法设计

### 5.1 核心循环

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  怪物生成 → 沿路径前进 → 炮塔显示目标字母                  │
│                          ↓                               │
│           玩家输入字母 → 判定正误                         │
│                  ↓正确          ↓错误                    │
│              发射子弹        输入框变红                   │
│                  ↓                                    │
│              命中扣血                                   │
│                  ↓                                    │
│         怪物死亡 → 获得金币 → 购买/放置防御塔             │
│                  ↓                                    │
│         全部消灭 → 胜利 → 下一关                         │
│                                                          │
│         怪物到达终点 → 扣生命 → 生命归零 → 失败            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5.2 打字机制详述

#### 5.2.1 目标锁定规则

- **默认策略**：锁定最靠近终点的怪物（威胁最大）
- **字母显示**：怪物头顶悬浮显示需要输入的字母/字母组合
- **多目标切换**：当前目标死亡后，自动切换到下一个最靠前的怪物
- **炮塔同步**：终点炮塔始终朝向当前锁定目标

#### 5.2.2 输入规则

- **单字母阶段（简单）**：每个怪物对应 1 个字母，输入正确即发射
- **多字母阶段（困难）**：每个怪物对应 2-3 个字母，需全部输入正确才发射
- **输错处理**：输入错误时，当前输入进度重置（针对多字母），输入框短暂变红
- **大小写**：不区分大小写，统一转为小写判定

#### 5.2.3 连击与奖励

| 连击数 | 伤害加成 | 特效 |
|--------|----------|------|
| 5 连击 | +10% 伤害 | 连击数字显示 |
| 10 连击 | +25% 伤害 | 屏幕边缘微光 |
| 20 连击 | +50% 伤害 | 子弹变大+拖尾 |
| 50 连击 | +100% 伤害 | 全屏闪光特效 |

### 5.3 防御塔机制详述

#### 5.3.1 塔类型与属性

| 塔名称 | 费用 | 伤害 | 攻速 | 射程 | 攻击类型 | 特殊效果 |
|--------|------|------|------|------|----------|----------|
| **箭塔** | 50 | 10 | 1.0次/秒 | 120 | 单体 | 无 |
| **魔法塔** | 100 | 25 | 0.5次/秒 | 100 | 范围（范围50） | 范围伤害 |
| **冰霜塔** | 80 | 5 | 0.8次/秒 | 110 | 单体 | 减速30%，持续2秒 |
| **狙击塔** | 150 | 80 | 0.3次/秒 | 200 | 单体 | 超远射程 |
| **金币塔** | 120 | 0 | - | - | - | 每5秒产生5金币 |

#### 5.3.2 放置规则

- 场景被划分为 **N × M** 的网格（单元格大小建议：40×40px）
- 路径占用的格子不可放置
- 已放置塔的格子不可重复放置
- 放置时鼠标悬停显示：
  - 绿色半透明 + 射程圈 → 可放置
  - 红色半透明 → 不可放置
- 右键 / ESC 键取消放置模式

#### 5.3.3 塔攻击逻辑

1. 每帧检测攻击范围内的所有怪物
2. 按照优先级选择目标（最靠近终点的怪物）
3. 攻击冷却结束后发射子弹/攻击
4. 子弹飞行命中后结算伤害和特效

### 5.4 怪物机制详述

#### 5.4.1 怪物类型

| 怪物名称 | 血量 | 速度 | 金币奖励 | 外观特征 |
|----------|------|------|----------|----------|
| **史莱姆** | 30 | 30px/s | 5 | 绿色圆球状，基础怪物 |
| **疾行者** | 20 | 60px/s | 8 | 瘦长型，移动迅速 |
| **重甲兵** | 100 | 20px/s | 15 | 身披铠甲，移动缓慢 |
| **巫师** | 50 | 25px/s | 12 | 法袍造型，召唤小弟（可选） |
| **Boss** | 500 | 15px/s | 100 | 巨型体型，每关最后出现 |

#### 5.4.2 路径移动

- 路径由一系列坐标点定义
- 怪物沿路径点依次移动
- 转弯时平滑转向（可选：直接转角或插值平滑）
- 到达最后一个路径点即算到达终点

### 5.5 经济系统

| 收入来源 | 说明 |
|----------|------|
| 击杀怪物 | 不同怪物给予不同金币 |
| 金币塔 | 定期产生金币 |
| 关卡奖励 | 通关后额外金币奖励（可选） |

| 支出项 | 说明 |
|--------|------|
| 购买防御塔 | 从道具栏购买放置 |
| 升级塔 | 提升塔属性（可选） |

### 6.2 视觉风格

- **整体风格**：卡通扁平化 2D，色彩明亮活泼
- **主色调**：绿色系（草地/自然）+ 蓝色系（魔法/科技）
- **UI 风格**：圆角卡片 + 轻微阴影，现代简洁
- **字体**：
  - 标题：圆润可爱的无衬线字体
  - 字母输入：等宽字体（Consolas / Monaco 类）

### 6.3 主要界面详细设计

#### 6.3.1 主菜单界面

- 居中展示游戏标题「盲打防线」
- 下方排列按钮：开始游戏、关卡选择、设置、帮助
- 背景：游戏场景的模糊/简化版本，增加动感

#### 6.3.2 关卡选择界面

- 网格布局展示关卡卡片
- 已通关关卡显示星数评分
- 未解锁关卡显示锁图标
- 每个关卡卡片显示：关卡编号、关卡名称、最佳成绩

#### 6.3.3 游戏主界面

**顶部状态栏**（从左到右）：
- 关卡编号 + 名称
- 金币数量（图标 + 数字）
- 生命值（心形图标 + 数字）
- 波次进度（x/y 格式）
- 暂停/设置按钮

**左侧道具栏**（从上到下）：
- 标题：「防御塔」
- 塔卡片列表（每个卡片：塔图标 + 名称 + 价格）
- 金币不足时卡片变灰，提示金币不足

**底部输入区**：
- 左侧：当前目标字母展示（大号字母，高亮当前需输入的那个）
- 中间：输入框（始终聚焦，视觉上有焦点状态）
- 右侧：实时数据（WPM + 准确率）

#### 6.3.4 放置模式界面

- 整个游戏场景叠加半透明网格
- 路径格子：红色半透明（禁止放置）
- 可放置格子：鼠标悬停时绿色高亮
- 鼠标跟随：塔的半透明预览 + 射程圈
- 顶部提示：「点击放置防御塔，右键取消」

#### 6.3.5 胜利/失败弹窗

**胜利弹窗**：
- 标题：「关卡胜利！」
- 统计数据：击杀数、打字速度、准确率、获得金币、用时
- 星级评价（1-3星）
- 按钮：下一关、重玩、返回菜单

**失败弹窗**：
- 标题：「防线失守...」
- 统计数据：击杀数、存活波次、打字数据
- 鼓励文案
- 按钮：重新挑战、返回菜单

### 6.4 动效设计

| 动效 | 描述 | 触发时机 |
|------|------|----------|
| 子弹飞行 | 子弹从炮塔飞向目标，带拖尾 | 正确输入字母时 |

| 怪物受击 | 怪物闪白 + 轻微抖动 | 怪物受到伤害时 |
| 怪物死亡 | 逐渐缩小/消散 + 金币弹出 | 怪物血量归零时 |

## 7. 关卡与数值设计

### 7.1 关卡设计原则

- 循序渐进：难度逐步提升，不出现陡增
- 教学融合：新机制在对应关卡首次出现
- 策略多样性：鼓励尝试不同的塔组合

### 7.2 关卡列表（前10关）

| 关卡 | 名称 | 波次 | 怪物类型 | 新机制 | 目标难度 |
|------|------|------|----------|--------|----------|
| 1 | 初识防线 | 3 | 史莱姆 | 教学关，单字母 | 简单 |
| 2 | 打字节奏 | 4 | 史莱姆 | 增加怪物数量 | 简单 |

## 8. 非功能需求

### 8.1 性能需求

| 指标 | 要求 |
|------|------|
| 帧率 | 稳定 60 FPS（同屏 50+ 怪物时不低于 30 FPS） |
| 加载时间 | 首屏加载 ≤ 3s，关卡切换 ≤ 1s |
| 内存占用 | ≤ 200MB |
| 兼容性 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ |

### 8.2 体验需求

| 指标 | 要求 |
|------|------|
| 操作延迟 | 按键到子弹发射的延迟 ≤ 50ms |
| 反馈及时性 | 每次输入都有即时视觉/听觉反馈 |
| 引导友好 | 新玩家 1 分钟内理解核心玩法 |
| 可暂停 | 游戏可随时暂停/继续 |

### 8.3 数据存储需求

- 使用 `localStorage` 存储：
  - 关卡进度
  - 每关最佳成绩（星数、分数）
  - 设置偏好（音量、难度等）
  - 累计统计数据

### 8.4 无障碍需求

- 颜色对比度符合 WCAG 2.1 AA 标准
- 重要信息不仅依赖颜色传达（如：扣血同时有震动+音效）
- 支持键盘操作菜单（Tab 切换，Enter 确认）

## 9. 技术实现方案

### 9.1 技术选型

| 模块 | 技术选择 | 选型理由 |
|------|----------|----------|
| **渲染引擎** | Phaser 3 | 成熟的 2D 游戏框架，内置物理、动画、碰撞检测，性能优秀 |
| **UI 框架** | React 18 | 组件化管理菜单、弹窗、道具栏等 DOM 界面 |
| **构建工具** | Vite | 极速热更新，开发体验好 |
| **语言** | TypeScript | 类型安全，利于维护复杂游戏逻辑 |
| **状态管理** | Zustand | 轻量级，管理游戏全局状态 |
| **样式** | Tailwind CSS | 快速构建 UI |

## 12. 风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|----------|
| **玩法趣味性不足** | 中 | 高 | 快速原型验证，根据反馈调整；加入连击、特殊怪等增加策略深度 |
| **Canvas 性能问题** | 中 | 中 | 提前做性能测试；实现对象池；控制同屏数量；分层渲染 |
| **打字与塔防平衡难** | 中 | 高 | 可调节难度；设计互补而非竞争的关系；塔是辅助，打字是核心 |
| **关卡设计工作量大** | 高 | 中 | 数据驱动配置；先做核心关卡再扩展；可用关卡编辑器工具 |
| **美术资源缺乏** | 高 | 中 | 用几何图形先做原型；使用免费素材网站；考虑AI辅助 |
| **难度曲线不合理** | 中 | 中 | 多轮测试调整；加入难度选择；动态难度调节 |
| **玩家打字水平差异大** | 高 | 中 | 分难度模式；从单字母渐进到多字母；提供练习模式 |
