## 1. 架构概述

### 1.1 设计原则

本架构设计遵循以下核心原则：

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个模块/类只负责一个功能领域，边界清晰 |
| **依赖倒置** | 高层模块不依赖低层模块，都依赖抽象接口 |
| **数据驱动** | 关卡、怪物、塔等配置全部数据化，便于调整和扩展 |
| **可扩展** | 打字引擎、塔类型、怪物类型通过插件化/策略模式扩展 |
| **可测试** | 核心逻辑与渲染层解耦，便于单元测试 |
| **性能优先** | 对象池、分层渲染、按需更新，确保 60FPS |

### 1.2 架构风格

- **分层架构**：UI 层 → 状态层 → 游戏逻辑层 → 渲染层
- **ECS 轻量化**：借鉴 ECS（实体-组件-系统）思想，但不过度设计
- **事件驱动**：模块间通过事件总线通信，降低耦合

### 1.3 约束条件

- **纯前端单机游戏**：无服务器，无网络请求
- **本地存储**：所有数据通过 `localStorage` / `IndexedDB` 持久化
- **浏览器兼容**：Chrome 90+、Firefox 88+、Safari 14+
- **性能目标**：同屏 50+ 怪物时稳定 30FPS 以上

## 2. 技术选型说明

### 2.1 技术栈总览

| 类别 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **开发语言** | TypeScript | 5.x | 类型安全，IDE 支持好，降低大型项目维护成本 |
| **构建工具** | Vite | 5.x | 极速 HMR，原生 ESM，构建速度快 |
| **游戏引擎** | Phaser | 3.70+ | 成熟的 2D 游戏框架，内置物理、动画、场景管理、资源加载 |
| **UI 框架** | React | 18.x | 组件化开发，生态丰富，适合菜单/弹窗/道具栏等 DOM UI |
| **状态管理** | Zustand | 4.x | 轻量级，API 简洁，无 boilerplate，适合游戏全局状态 |
| **样式方案** | Tailwind CSS | 3.x | 原子化 CSS，快速构建 UI，与 React 配合良好 |
| **动画库** | framer-motion（可选） | 11.x | React 组件动画，用于菜单转场、弹窗动效 |
| **测试框架** | Vitest | 1.x | Vite 原生支持，速度快，用于核心逻辑单元测试 |
| **代码规范** | ESLint + Prettier | - | 统一代码风格，提前发现问题 |

### 2.2 为什么不用其他方案

| 方案 | 不采用原因 |
|------|-----------|
| 纯 Canvas 自绘 | 重复造轮子，Phaser 已提供完善的游戏基础设施 |
| PixiJS | 渲染性能优秀，但缺少场景管理、物理等游戏必备能力 |
| Unity / Godot | 重引擎，网页加载慢，对于 2D 塔防过度设计 |
| Vue | 与 React 类似，团队更熟悉 React 生态 |
| Redux | 过于重型，游戏状态不需要时间旅行等特性 |

```
│  └──────────┴──────────┴──────────┴──────────┴──────────┘    │
├──────────────────────────────────────────────────────────────┤
│                       状态管理层 (Zustand)                    │
│  ┌─────────────┬──────────────┬──────────────────────────┐   │
│  │  游戏状态   │   用户数据   │      设置/配置            │   │
│  └─────────────┴──────────────┴──────────────────────────┘   │
├──────────────────────────────────────────────────────────────┤
│                        游戏逻辑层                              │
│  ┌──────────────┬──────────────┬──────────────────────────┐  │
│  │  打字引擎    │   战斗系统   │     关卡/波次管理        │  │
│  └──────────────┴──────────────┴──────────────────────────┘  │
│  ┌──────────────┬──────────────┬──────────────────────────┐  │
│  │  怪物系统    │   防御塔系统  │     子弹/碰撞系统        │  │
│  └──────────────┴──────────────┴──────────────────────────┘  │
├──────────────────────────────────────────────────────────────┤
│                        渲染层 (Phaser)                         │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐    │
│  │  场景管理 │  精灵渲染 │  动画系统 │  物理/碰撞 │  资源管理 │    │
│  └──────────┴──────────┴──────────┴──────────┴──────────┘    │
└──────────────────────────────────────────────────────────────┘
         ↕ 事件总线 (EventBus) 贯穿各层通信
```

### 3.2 模块依赖关系

```
React UI 组件
    ↓ (订阅)
Zustand Store
    ↓ (调用 / 订阅)
游戏管理器 (GameManager)
    ├── 场景管理器 (SceneManager)
    ├── 打字引擎 (TypingEngine)
    ├── 关卡管理器 (LevelManager)
    ├── 怪物管理器 (MonsterManager)
    ├── 防御塔管理器 (TowerManager)
    ├── 子弹管理器 (BulletManager)
    └── 事件总线 (EventBus)
```

### 3.3 核心流程：玩家输入 → 子弹发射

```
用户按键
  ↓
React 输入框组件 (捕获 keydown)
  ↓
TypingEngine.input(char)  ←── 策略模式：QwertyEngine / ShuangpinEngine / WubiEngine
  ↓
判定结果（正确/错误/完成）
  ↓
EventBus.emit('typing:complete')
  ↓
GameManager 监听 → 调用炮塔发射子弹
  ↓
BulletManager 创建子弹实体
  ↓
Phaser 渲染子弹飞行动画
  ↓
碰撞检测（每帧）
  ↓
命中 → 怪物扣血 → 死亡判定 → 金币奖励
  ↓
EventBus.emit('monster:kill')
  ↓
Zustand Store 更新金币、击杀数
  ↓
React UI 重新渲染显示
```

### 4.2 目录设计说明

| 目录 | 职责 | 依赖 |
|------|------|------|
| `src/game/` | 纯游戏逻辑，与渲染框架无关 | 只依赖类型和工具 |
| `src/phaser/` | Phaser 渲染实现，将 game 层的实体渲染出来 | 依赖 `src/game/` |
| `src/store/` | Zustand 全局状态，连接 UI 和游戏逻辑 | 依赖 `src/game/` 类型 |
| `src/components/` | React UI 组件 | 依赖 `src/store/` |
| `src/storage/` | 本地持久化封装 | 独立，被 store 调用 |

**关键设计**：`src/game/` 不依赖 Phaser 或 React，是纯逻辑层。这样既方便测试，也为未来更换渲染引擎留有余地。

### 5.2 事件总线 (EventBus)

**职责**：模块间解耦通信

```typescript
// 事件类型定义
interface GameEvents {
  'game:start': void;
  'game:pause': void;
  'game:resume': void;
  'game:victory': VictoryData;
  'game:defeat': DefeatData;

  'monster:spawn': Monster;
  'monster:hit': { monster: Monster; damage: number };
  'monster:death': Monster;
  'monster:reachEnd': Monster;

  'tower:place': Tower;
  'tower:sell': Tower;
  'tower:attack': { tower: Tower; target: Monster };

  'bullet:fire': Bullet;
  'bullet:hit': { bullet: Bullet; monster: Monster };

  'typing:correct': { char: string; combo: number };
  'typing:wrong': { char: string };
  'typing:complete': { target: TypingTarget; combo: number };
  'typing:targetChange': TypingTarget;

  'gold:change': { current: number; delta: number };
  'life:change': { current: number; delta: number };
  'wave:start': number;
  'wave:complete': number;
}

class EventBus {
  public on<K extends keyof GameEvents>(
    event: K,
    handler: (data: GameEvents[K]) => void
  ): () => void;  // 返回取消订阅函数

  public emit<K extends keyof GameEvents>(
    event: K,
    data: GameEvents[K]
  ): void;
}
```

```typescript
abstract class Tower extends GameEntity {

}

// 具体塔类型

class TowerFactory {
```

### 5.4 系统层

#### 5.4.1 怪物系统

```typescript
class MonsterSystem {
  private monsters: Monster[] = [];
  private pool: ObjectPool<Monster>;  // 对象池
```

```typescript
class QwertyEngine implements ITypingEngine {
  public readonly type: TypingEngineType = 'qwerty';

  private dictionary: string[] = [];  // 字母/单词库

  public async loadDictionary(levelId?: number): Promise<void> {
    // 根据关卡加载不同难度的词库
    const dict = await import(`/assets/data/typing/qwerty/level${levelId}.json`);
    this.dictionary = dict.words;
  }

  public generateTarget(difficulty: number, levelId?: number): TypingTarget {
    // 根据难度决定长度：
    // 1-3: 单字母
    // 4-6: 双字母
    // 7-10: 三字母/短单词
    const length = this.getLengthByDifficulty(difficulty);
    const text = this.randomText(length);

    return {
      id: generateId(),
      engineType: 'qwerty',
      displayText: text,
      inputSequence: text.toLowerCase(),
      currentIndex: 0,
      isComplete: false,
      difficulty,
    };
  }

  public inputChar(target: TypingTarget, char: string): TypingInputResult {
    const lowerChar = char.toLowerCase();
    const expectedChar = target.inputSequence[target.currentIndex];
    const isCorrect = lowerChar === expectedChar;

    if (isCorrect) {
      target.currentIndex++;
      target.isComplete = target.currentIndex >= target.inputSequence.length;
    } else {
      // 错误时重置进度（可配置是否重置）
      // target.currentIndex = 0;  // 严格模式：错了重来
      // 宽松模式：不重置，只是不前进
    }

    return {
      success: isCorrect,
      complete: target.isComplete,
      progress: target.currentIndex / target.inputSequence.length,
      currentIndex: target.currentIndex,
      typedChar: char,
      correctChar: expectedChar,
    };
  }

  public getDisplayText(target: TypingTarget): TypingDisplayText {
    const seq = target.displayText;
    return {
      before: seq.slice(0, target.currentIndex),
      current: seq[target.currentIndex] || '',
      after: seq.slice(target.currentIndex + 1),
    };
  }

  public resetProgress(target: TypingTarget): void {
    target.currentIndex = 0;
    target.isComplete = false;

    return 4;
```

### 6.7 打字管理器（面向业务）

```typescript
// src/game/typing/TypingManager.ts

/**
 * 打字管理器
 * 业务层通过它使用打字功能，不直接接触具体引擎
 */
class TypingManager {
  private engine: ITypingEngine;
  private currentTarget: TypingTarget | null = null;
  private combo: number = 0;
  private totalKeystrokes: number = 0;
  private correctKeystrokes: number = 0;

  constructor(engineType: TypingEngineType = 'qwerty') {
    this.engine = EngineFactory.getEngine(engineType);
  }

  /** 切换打字引擎 */
  public switchEngine(type: TypingEngineType): void {
    this.engine = EngineFactory.getEngine(type);
    this.reset();
  }

  /** 生成新的打字目标 */
  public newTarget(difficulty: number, levelId?: number): TypingTarget {
    this.currentTarget = this.engine.generateTarget(difficulty, levelId);
    return this.currentTarget;
  }

  /** 处理输入 */
  public handleInput(char: string): TypingResult {
    if (!this.currentTarget) {
      return { success: false, complete: false, combo: 0 };
    }

    this.totalKeystrokes++;
    const result = this.engine.inputChar(this.currentTarget, char);

    if (result.success) {
      this.correctKeystrokes++;
      if (result.complete) {
        this.combo++;
      }
    } else {
      this.combo = 0;
    }

    return {
      ...result,
      combo: this.combo,
    };
  }

  /** 获取当前目标的显示文本 */
  public getDisplayText(): TypingDisplayText | null {
    if (!this.currentTarget) return null;
    return this.engine.getDisplayText(this.currentTarget);
  }

  /** 统计数据 */
  public getStats(): TypingStats {
    return {
      combo: this.combo,
      accuracy: this.totalKeystrokes > 0
        ? this.correctKeystrokes / this.totalKeystrokes
        : 1,
      totalKeystrokes: this.totalKeystrokes,
      correctKeystrokes: this.correctKeystrokes,
    };
  }

  public reset(): void {
    this.currentTarget = null;
    this.combo = 0;
    this.totalKeystrokes = 0;
    this.correctKeystrokes = 0;
  }
}

interface TypingResult extends TypingInputResult {
  combo: number;
}

interface TypingStats {
  combo: number;
  accuracy: number;
  totalKeystrokes: number;
  correctKeystrokes: number;
}
```

```typescript
/**
 * 本地存储服务
 * 封装 localStorage 操作，提供类型安全的读写
 */
class StorageService {
  private static prefix = 'typing_defense_';

  /** 保存数据 */
  public static set<T>(key: StorageKey, value: T): void {
    try {
      const fullKey = this.prefix + key;
      const json = JSON.stringify(value);
      localStorage.setItem(fullKey, json);
    } catch (e) {
      console.error('Storage set error:', e);
    }
  }

  /** 读取数据 */
  public static get<T>(key: StorageKey, defaultValue: T): T {
    try {
      const fullKey = this.prefix + key;
      const json = localStorage.getItem(fullKey);
      if (json === null) return defaultValue;
      return JSON.parse(json) as T;
    } catch (e) {
      console.error('Storage get error:', e);
      return defaultValue;
    }
  }

  /** 删除数据 */
  public static remove(key: StorageKey): void {
    const fullKey = this.prefix + key;
    localStorage.removeItem(fullKey);
  }

  /** 清空所有数据 */
  public static clear(): void {
    Object.values(StorageKey).forEach(key => {
      localStorage.removeItem(this.prefix + key);
    });
  }

  /** 检查是否有存档 */
  public static hasSave(): boolean {
    return localStorage.getItem(this.prefix + StorageKey.GameSave) !== null;
  }
}
```

## 9. 性能优化方案

### 9.1 对象池（Object Pool）

**应用场景**：怪物、子弹、特效粒子等频繁创建销毁的对象

```typescript
// src/game/utils/pool.ts

class ObjectPool<T extends { reset(): void; active: boolean }> {
  private pool: T[] = [];
  private factory: () => T;
  private maxSize: number;

  constructor(factory: () => T, initialSize: number = 20, maxSize: number = 100) {
    this.factory = factory;
    this.maxSize = maxSize;
    // 预创建
    for (let i = 0; i < initialSize; i++) {
      this.pool.push(factory());
    }
  }

  public acquire(): T {
    const obj = this.pool.find(o => !o.active);
    if (obj) {
      obj.active = true;
      return obj;
    }
    // 池满，创建新的
    if (this.pool.length < this.maxSize) {
      const newObj = this.factory();
      newObj.active = true;
      this.pool.push(newObj);
      return newObj;
    }
    // 达到上限，复用最旧的
    const oldest = this.pool[0];
    oldest.reset();
    oldest.active = true;
    return oldest;
  }

  public release(obj: T): void {
    obj.active = false;
    obj.reset();
  }

  public releaseAll(): void {
    this.pool.forEach(obj => {
      obj.active = false;
      obj.reset();
    });
  }
}
```

### 9.2 分层渲染

### 9.4 其他优化策略

| 优化点 | 方案 |
|--------|------|
| **更新频率控制** | 塔的目标查找每 N 帧执行一次（不用每帧） |
| **离屏剔除** | 视口外的对象跳过渲染（Phaser 内置支持） |
| **纹理图集** | 精灵图打包成 Sprite Sheet，减少 Draw Call |
| **静态批处理** | 背景、路径等静态元素合并渲染 |

| **配置缓存** | JSON 配置加载后缓存到内存，不重复读取 |
| **节流防抖** | UI 更新（如数字跳动）节流到每 100ms 一次 |

## 11. 关键技术难点与解决方案

### 11.1 React 与 Phaser 的集成

**难点**：React 管理 DOM UI，Phaser 管理 Canvas 渲染，两者如何协同？

**方案**：
1. **Phaser 作为 React 组件**：封装 `PhaserGame` 组件，在 `useEffect` 中初始化 Phaser 实例
2. **状态共享**：通过 Zustand Store 作为中间层，Phaser 内的逻辑更新 Store，React 订阅 Store
3. **事件通信**：Phaser 内通过 `EventBus` 发事件，React 组件监听并响应
4. **Canvas 定位**：Phaser Canvas 使用绝对定位，React UI 覆盖在 Canvas 上层

```typescript
// PhaserGame 组件示例
const PhaserGame: React.FC = () => {
  const containerRef = useRef<HTMLDivElement>(null);
  const gameRef = useRef<Phaser.Game | null>(null);

  useEffect(() => {
    if (!containerRef.current) return;

    const config: Phaser.Types.Core.GameConfig = {
      type: Phaser.AUTO,
      parent: containerRef.current,
      width: GAME_WIDTH,
      height: GAME_HEIGHT,
      scene: [BootScene, GameScene],
      // ...
    };

    gameRef.current = new Phaser.Game(config);

    return () => {
      gameRef.current?.destroy(true);
    };
  }, []);

  return <div ref={containerRef} className="phaser-container" />;
};
```

### 11.2 打字输入的焦点管理

**难点**：游戏中既要监听全局键盘输入，又不能干扰输入框和菜单操作

**方案**：
- 游戏进行时：隐藏真实 input，全局监听 `keydown` 事件
- 输入框显示：用一个视觉输入框 + 内部状态维护，无实际焦点
- 菜单/弹窗时：暂停游戏，启用正常的输入框焦点

### 11.3 路径移动与转弯

**难点**：怪物沿多段路径移动，转弯处平滑过渡

**方案**：
- 路径由 `Point[]` 定义
- 怪物记录当前路径段索引和段内进度
- 每帧根据速度计算前进距离，段间切换时处理剩余距离
- 转弯时：直接转角（简单）或 弧形插值（平滑）

### 11.4 碰撞检测精度

**难点**：子弹速度快，可能出现"穿透"（一帧飞过目标）

**方案**：
- 连续碰撞检测（CCD）：子弹移动后检测起点-终点线段与目标的碰撞
- 简化：子弹速度不超过目标半径的两倍（当前游戏节奏下足够）
